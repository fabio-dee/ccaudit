## Accomplishments

- **SC4 locked — entry preflight before picker.** In `runInteractiveGhostFlow`, inserted a 44-line block between the TTY guard (`case 'ok': break;`) and the empty-inventory short-circuit. The block runs `detectClaudeProcesses(process.pid, defaultProcessDeps)`:
  - spawn-failed → `"Could not verify Claude Code is stopped: <error>"` + `"Run from a clean shell where ps (Unix) or tasklist (Windows) is available."` stderr + `process.exitCode = 2` + return.
  - zero processes → fall through to normal flow unchanged.
  - one or more processes → classify self-invocation via `walkParentChain`, build `initialResult`, invoke `runPreflightRetryLoop({ phase: 'entry' })`:
    - `cancelled` → `"No changes made."` stderr + exit 0. **No checkpoint written** (INV-S2 compatible — the block runs BEFORE `writeCheckpoint`).
    - `spawn-failed` → same copy pair as above + exit 2.
    - `clear` → Claude was closed during the retry prompt. Fall through.
- **SC5b locked — bust-time retry preserves selectedItems identity.** Changed `const result = await runBust({ ... })` → `let result = await runBust({ ... })`, then wrapped in `while (result.status === 'running-process') { … }`. Inside the loop, `runPreflightRetryLoop({ phase: 'bust', initialResult: { selfInvocation, pids } })` handles the three outcomes (same as entry path), and on `clear` re-invokes `runBust` with the **SAME** `selectedItems` const captured at line 235 — verbatim, no rebinding, no `new Set(...)` wrapping. The hash-mismatch / success / partial-success render blocks are byte-unchanged (W6 minimal-diff guardrail).
- **SC5 locked — single source of truth for preflight copy.** The non-interactive `case 'running-process':` arm at ~L1315 (previously 20 lines of inline `console.error` calls) is replaced with a single `process.stderr.write(renderRunningProcessMessage({ selfInvocation: result.selfInvocation, pids: result.pids }))`. Byte-identical stderr is enforced by the `_preflight-copy.ts` inline-snapshot test from plan 02 (`it('matches a deterministic inline snapshot for selfInvocation=true (SC5 drift guard)')`).
- **Barrel surface expanded.** `packages/internal/src/index.ts` now re-exports `detectClaudeProcesses` and `walkParentChain` at the top-level `@ccaudit/internal` barrel — previously these were only available via the `remediation/index.ts` subpath. The CLI layer can now import them alongside `runBust` / `defaultProcessDeps` in a single flat block. Zero-runtime-deps invariant untouched (both symbols were already bundled; only their export visibility changed).
- **Test fix preserves Phase 3.1 coverage.** `apps/ccaudit/src/__tests__/tabbed-picker-terminal-too-short.test.ts` was asserting that the D3.1-16 height gate fires BEFORE any `ps` lookup (its comment explicitly said "No fake-ps shim is needed"). Phase 3.2's entry preflight runs `ps` BEFORE `selectGhosts` (which is where the height gate lives), so without the shim the subprocess fails closed with exit 2. Fix: added `buildFakePs(tmpHome)` in `beforeEach` and wrapped the `describe` in `skipIf(process.platform === 'win32')` — same pattern as the other ps-dependent integration tests. The D3.1-16 assertion (exitCode=1 + exact stderr + no manifest) is preserved unchanged.
