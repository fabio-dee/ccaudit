## Decisions Made

- **Byte-identical copy at both call sites.** The plan's interface contract (lines 114-141) locked to keep interactive and non-interactive stderr bytes identical. Updated wording would require touching both paths in a future phase; the SC5 byte-equality integration test enforces this. No wording drift for `--dangerously-bust-ghosts` vs `ccaudit ghost --interactive`.
- **Self-invocation never retries.** `initialResult.selfInvocation === true` prints the copy and returns `{ status: 'cancelled' }` without invoking `detectFn` or prompting — closing the parent Claude Code session would kill ccaudit itself. This is the only branch that emits a stderr write without a prompt.
- **Trailing-newline contract.** Existing `console.error` calls appended `\n` per line. The helper's `lines.join('\n') + '\n'` yields byte-identical stderr when written via `process.stderr.write(rendered)` (a single call). Inline-snapshot test locks this.
- **`isCancel` DI shape.** Typed as `(value: unknown) => boolean` (not `value is symbol`) so `vi.fn()` mocks satisfy the interface without declaring a type predicate. Mirrors confirmation.ts lines 163-166.
- **W2 single-emit spy guard.** Added a spy-on-`process.stderr.write` test that counts writes containing the identifying "Don't cross the streams!" phrase to prove exactly one emit per loop iteration. Guards against a future refactor accidentally double-printing the copy.
