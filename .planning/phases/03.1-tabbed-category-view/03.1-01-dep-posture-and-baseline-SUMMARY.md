## Decisions Made

- **Timing of baseline capture:** Baseline was captured **after** the `@clack/core` pin, not before. Rationale: the pin does not add any new bundle content (`@clack/core@1.2.0` was already bundled transitively — the build output now just _labels_ it as a direct bundled dep). Capturing post-pin means Plan 05's <10 KB gate measures only the `TabbedGhostPicker` subclass's incremental cost, which is exactly what D3.1-16 is meant to protect against. If we had captured pre-pin, the delta in Plan 05 could include any incidental resolution quirks from the pin itself; capturing post-pin eliminates that variable.
- **apps/ccaudit untouched:** Per D3.1-14 the subclass lives in `packages/terminal/src/tui/tabbed-picker.ts`, so only the terminal workspace needs the explicit devDep. Adding one to `apps/ccaudit` would not break anything but would be redundant. Followed the plan's Step 3 guidance to "leave it untouched to preserve the zero-runtime-deps invariant."
- **Gzip level 9 reuse:** `apps/ccaudit/scripts/bundle-size-check.mjs` uses `gzipSync(buf, { level: 9 })`. The baseline command in the plan uses the same invocation so Plan 05 can compute `actual = gzipSync(dist/index.js, { level: 9 }).length` and subtract `170619` without algorithm drift.
