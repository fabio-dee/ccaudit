## Decisions Made

1. **`CCAUDIT_TEST_STDOUT_ROWS` env seam chosen over pty / LINES** — verified live that `process.stdout.rows` is `undefined` under piped stdio and Node does NOT honour `LINES` for non-TTY streams. The env-var override is the simplest deterministic mechanism that preserves the zero-runtime-deps invariant. Plan 04 Task 5 explicitly authorised this alternative.
2. **`stdin.end()` after confirm keys** — @clack/prompts registers a keypress listener that survives the picker's submit(); without `end()` the subprocess hangs indefinitely. This is the fundamental sub-agent-of-the-lifecycle pattern for any future tests that drive the full interactive bust.
3. **Cross-tab test in-source, not separate file** — plan explicitly preferred in-source when feasible. The class's public action methods already provide the test seam; no `__testDispatchKey` seam was needed.
4. **Assertions accept both Unicode and ASCII glyphs** — NO_COLOR=1 in the subprocess env flips `shouldUseAscii()` to `true`; both output paths must be covered for the test to run reliably in any CI environment.
5. **Per-test 25s timeout for the Tab/ArrowRight tests** — the full interactive bust pipeline (picker ready + key sequence + confirmation + bust execution) takes 1–2s wall time plus key-dispatch settle waits; vitest's 5s default was insufficient and produced 20-second timeouts before stdin.end() was added.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Terminal-too-short gate unreachable from piped subprocess**

- **Found during:** Task 5 (terminal-too-short gate test).
- **Issue:** `process.stdout.rows` is `undefined` when stdout is a pipe (subprocess context), and Node's readline/tty module does NOT honour the `LINES` env var for non-TTY streams. The gate's `stdoutRows !== undefined && stdoutRows < 14` check short-circuits on undefined — so the subprocess opens the picker instead of exiting 1 with the D3.1-16 message.
- **Fix:** Wired a `CCAUDIT_TEST_STDOUT_ROWS` env-var override into `select-ghosts.ts`'s D3.1-16 gate (regex-guarded to numeric input). Plan 04 Task 5 explicitly authorised this alternative ("if `LINES` fails: add a test-only env var `CCAUDIT_TEST_STDOUT_ROWS` ...").
- **Files modified:** `packages/terminal/src/tui/select-ghosts.ts`.
- **Verification:** Direct subprocess invocation with `CCAUDIT_TEST_STDOUT_ROWS=10` exits 1 with the exact D3.1-16 message (verified via ad-hoc `/tmp/test-short2.mjs`); in-source `select-ghosts.ts` tests continue to pass 4/4.
- **Committed in:** 3f577f5 (Task 5 commit — the seam and its test ship together).

**2. [Rule 3 - Blocking] Subprocess hangs after interactive bust completes**

- **Found during:** Task 4 first run (20-second vitest timeouts despite manifest being written correctly).
- **Issue:** After the bust renders its success box, the Node event loop stays alive because `@clack/core` + `@clack/prompts` leave a stdin keypress listener registered. The subprocess hangs until SIGKILL.
- **Fix:** Call `spawned.child.stdin.end()` after sending the confirm sequence in `runPickerWithTabKey`. This closes stdin so the listener releases the event loop and the subprocess exits naturally with its already-set `process.exitCode = 0`.
- **Files modified:** `apps/ccaudit/src/__tests__/tabbed-picker-tab-nav-keys.test.ts` (in the shared `runPickerWithTabKey` helper).
- **Verification:** Test duration dropped from 20s timeout → 1–2s pass.
- **Committed in:** f8bc214 (Task 4 commit — the helper and its tests ship together).

**3. [Rule 1 - Bug] Eslint `no-control-regex` errors in overflow test**

- **Found during:** Post-Task-5 `pnpm lint` in the verify chain.
- **Issue:** Three per-line `eslint-disable-next-line no-control-regex` comments in `tabbed-picker-overflow.test.ts` were mis-placed — the first landed on `return s` (which has no regex) so eslint reported it as an unused directive AND fired on the first regex literal.
- **Fix:** Replaced with a single block-level `/* eslint-disable no-control-regex -- ANSI stripping requires literal \x1b */` wrapping the helper function.
- **Files modified:** `apps/ccaudit/src/__tests__/tabbed-picker-overflow.test.ts`.
- **Verification:** `pnpm lint` now exits 0.
- **Committed in:** fdbb821 (combined fix-up commit).

**4. [Rule 1 - Bug] TS2339 on `manifest.ops.map(o => o.category)` in tab-nav-keys test**

- **Found during:** Post-Task-5 `pnpm -r typecheck`.
- **Issue:** `ManifestOp` is a discriminated union (Archive | Disable | Flag | Refresh | Skipped); only `ArchiveOp` and `SkippedOp` carry a `category` field. Direct `.map(o => o.category)` on the full union fails TS2339.
- **Fix:** Filter ops by `op_type === 'archive'` first, then read `category` from the narrowed type. The Plan 04 fixture only produces archive ops (2 agents + 2 skills → selection archives 1+1), so the filter is correct, not a workaround.
- **Files modified:** `apps/ccaudit/src/__tests__/tabbed-picker-tab-nav-keys.test.ts`.
- **Verification:** `pnpm -r typecheck` now exits 0.
- **Committed in:** fdbb821 (combined fix-up commit).

**5. [Rule 1 - Lint] Oxfmt formatting mismatch in tab-nav-keys test**

- **Found during:** Post-Task-5 `pnpm format:check`.
- **Issue:** The `it()` block signature with numeric timeout as the third arg needed oxfmt-driven line wrapping.
- **Fix:** Ran `pnpm format` (oxfmt --write).
- **Files modified:** `apps/ccaudit/src/__tests__/tabbed-picker-tab-nav-keys.test.ts`.
- **Verification:** `pnpm format:check` now exits 0.
