<objective>
Close the escaped gap discovered during the Phase 7 manual smoke test: `ccaudit --dry-run` crashes with unhandled `ENOENT` when `~/.claude/skills/` contains a broken symlink (or any path that cannot be stat'd).

Purpose: The automated Phase 7 verification gate passed (353/353 tests, 93.49% stmts, gsd-verifier `passed`) but the very first real-world execution against a live `~/.claude/` installation crashed because the synthetic integration-test fixture never exercised the "inventory references a path that cannot be stat'd" condition. Two layers of the code pipeline both trust that `readdir`/`glob` results can always be stat'd later: (1) `scanSkills`/`scanAgents` do not populate `mtimeMs` at discovery time, (2) `computeGhostHash` falls back to an unprotected `stat()` call when `mtimeMs` is undefined. Any `ENOENT`/`ELOOP`/`EACCES`/`ENOTDIR` in that fallback crashes the entire `Promise.all` inside `computeGhostHash` and the CLI exits with a stack trace before `writeCheckpoint` is called — DRYR-02 and DRYR-03 both silently fail on real inventories.

Output:
- `scanSkills` and `scanAgents` return items with `mtimeMs: number` populated; broken symlinks and missing files are silently excluded (mirrors the Phase 3 `scanMemoryFiles` pattern at `scan-memory.ts:44-56`)
- `computeGhostHash` has a defensive safety net: the async mapper returns `HashRecord | null`, wraps the `statFn` call in try/catch, and filters nulls before the canonical sort/serialize step (frozen D-17 contract is preserved — "items enter/leave eligible set" covers this)
- New in-source and integration tests that would have caught this bug in CI: a `scan-skills` broken-symlink test, a `scan-agents` missing-file race test, a `computeGhostHash` un-stat-able-item test, and an end-to-end `dry-run-command` regression test with a broken-symlink skill fixture
- Full workspace suite green (>= 353 tests) and coverage gate still at or above Phase 6 thresholds (80/70/80/80)

This is a surgical gap-closure plan. Scope is strictly limited to the four files listed in `files_modified`. No refactoring of unrelated code. No touching REQUIREMENTS.md traceability table. No Phase 8 work. No additional features.
