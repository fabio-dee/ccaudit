    ```markdown
    ## `--dangerously-bust-ghosts` JSON envelope

    When `--dangerously-bust-ghosts` is run with `--json` (or `--ci` which implies it), the output is wrapped in the same top-level envelope as other commands:

    ```json
    {
      "meta": {
        "command": "ghost",
        "version": "<ccaudit version>",
        "since": "7d",
        "timestamp": "<iso-8601>",
        "exitCode": 0
      },
      "bust": { ... }
    }
    ```

    The `bust` payload is a discriminated union keyed by `status`. Every possible outcome:

    ### Success variants

    `status: "success"` — all ops completed cleanly.
    ```json
    {
      "status": "success",
      "manifestPath": "/home/user/.claude/ccaudit/manifests/bust-2026-04-05T18-30-00Z.jsonl",
      "counts": {
        "archive": { "completed": 3, "failed": 0 },
        "disable": { "completed": 1, "failed": 0 },
        "flag":    { "completed": 2, "failed": 0, "refreshed": 0, "skipped": 0 }
      },
      "duration_ms": 145
    }
    ```

    `status: "partial-success"` — some ops failed; see manifest for details.
    ```json
    {
      "status": "partial-success",
      "manifestPath": "...",
      "counts": { ... },
      "failed": 2,
      "duration_ms": 145
    }
    ```

    ### Gate failures

    `status: "checkpoint-missing"` — no prior `--dry-run` checkpoint.
    ```json
    { "status": "checkpoint-missing", "checkpointPath": "/home/user/.claude/ccaudit/.last-dry-run" }
    ```

    `status: "checkpoint-invalid"` — checkpoint exists but is unparseable / wrong version / missing fields.
    ```json
    { "status": "checkpoint-invalid", "reason": "parse-error: Unexpected token" }
    ```

    `status: "hash-mismatch"` — inventory changed since last dry-run.
    ```json
    {
      "status": "hash-mismatch",
      "expected": "sha256:<old-hash>",
      "actual":   "sha256:<new-hash>"
    }
    ```

    ### Preflight failures

    `status: "running-process"` — Claude Code is currently running.
    ```json
    {
      "status": "running-process",
      "pids": [12345, 67890],
      "selfInvocation": false,
      "message": "Claude Code is running (pids: 12345, 67890). Close all Claude Code windows and re-run..."
    }
    ```

    When `selfInvocation` is `true`, the user was running ccaudit from inside a Claude Code session (Bash tool). The `message` field names the pid in the parent chain and instructs the user to open a standalone terminal.

    `status: "process-detection-failed"` — `ps` or `tasklist` could not be spawned.
    ```json
    { "status": "process-detection-failed", "error": "ENOENT: ps not found on PATH" }
    ```

    ### User / config errors

    `status: "user-aborted"` — user declined at a prompt.
    ```json
    { "status": "user-aborted", "stage": "prompt1" }
    ```
    `stage` is `"prompt1"` (y/N) or `"prompt2"` (typed phrase).

    `status: "config-parse-error"` — `~/.claude.json` could not be parsed.
    ```json
    { "status": "config-parse-error", "path": "/home/user/.claude.json", "error": "Unexpected token" }
    ```

    `status: "config-write-error"` — atomic write of `~/.claude.json` (or manifest) failed.
    ```json
    { "status": "config-write-error", "path": "/home/user/.claude.json", "error": "EPERM" }
    ```

    ### Exit code mapping

    | `bust.status`              | Process exit code |
    |----------------------------|-------------------|
    | `success`, `user-aborted`  | 0                 |
    | `partial-success`          | 1                 |
    | `checkpoint-missing`       | 1                 |
    | `checkpoint-invalid`       | 1                 |
    | `hash-mismatch`            | 1                 |
    | `config-parse-error`       | 1                 |
    | `config-write-error`       | 1                 |
    | `running-process`          | 3                 |
    | `process-detection-failed` | 3                 |
    | *(non-TTY without `--yes-proceed-busting`, no JSON emitted)* | 4 |

    Note: exit code 4 (non-TTY without bypass) is emitted BEFORE the bust pipeline runs, so no JSON envelope is produced. The error is written to stderr only.
    ```
