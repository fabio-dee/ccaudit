Initialize request (client -> server):
```json
{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"ccaudit","version":"0.0.1"}}}
```

Initialize response (server -> client):
```json
{"jsonrpc":"2.0","id":1,"result":{"protocolVersion":"2025-06-18","capabilities":{"tools":{}},"serverInfo":{"name":"...","version":"..."}}}
```

Initialized notification (client -> server):
```json
{"jsonrpc":"2.0","method":"notifications/initialized"}
```

tools/list request (client -> server):
```json
{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}
```

tools/list response (server -> client):
```json
{"jsonrpc":"2.0","id":2,"result":{"tools":[{"name":"...","description":"...","inputSchema":{...}}]}}
```

    ```typescript
    export interface McpServerConfig {
      command: string;
      args?: string[];
      env?: Record<string, string>;
      type?: string; // 'stdio' | 'http' | 'sse' -- only stdio supported
    }

    export interface McpToolDefinition {
      name: string;
      description?: string;
      inputSchema?: Record<string, unknown>;
    }
    ```

    **Exported functions:**

    `listMcpTools(config: McpServerConfig, timeoutMs?: number): Promise<McpToolDefinition[]>`

    Implementation:
    1. Default timeoutMs = 15_000 (15 seconds)
    2. Check config.type -- if present and not 'stdio', throw: `Live measurement not available for ${config.type} transport. Use estimate.`
    3. Spawn child process: `spawn(config.command, config.args ?? [], { stdio: ['pipe', 'pipe', 'pipe'], env: { ...process.env, ...config.env } })`
    4. Set up readline interface on child.stdout for newline-delimited JSON parsing
    5. Set up timeout: `setTimeout(() => { child.kill(); reject(new Error('MCP server timed out ...')) }, timeoutMs)`
    6. Send initialize request: write JSON-RPC message + '\n' to child.stdin
       - `{ jsonrpc: '2.0', id: 1, method: 'initialize', params: { protocolVersion: '2025-06-18', capabilities: {}, clientInfo: { name: 'ccaudit', version: '0.0.1' } } }`
    7. Wait for line on stdout. Parse JSON. If id === 1, that is the initialize response.
    8. Send initialized notification: `{ jsonrpc: '2.0', method: 'notifications/initialized' }`
    9. Send tools/list request: `{ jsonrpc: '2.0', id: 2, method: 'tools/list', params: {} }`
    10. Wait for line on stdout with id === 2. Parse result.tools as McpToolDefinition[].
    11. Clear timeout, kill child process, close readline, resolve with tools array.
    12. Handle errors: child 'error' event (command not found), child 'exit' with nonzero code, parse errors. All reject with descriptive messages. Always clear timeout and kill child on error.

    `measureMcpTokens(config: McpServerConfig, timeoutMs?: number): Promise<{ tokens: number; confidence: 'measured'; source: string; toolCount: number }>`

    Implementation:
    1. Call `listMcpTools(config, timeoutMs)`
    2. For each tool: `(tool.name + (tool.description ?? '') + JSON.stringify(tool.inputSchema ?? {})).length`
    3. Sum all lengths, divide by 4, Math.ceil for token count
    4. Return `{ tokens, confidence: 'measured', source: 'live measurement via tools/list', toolCount: tools.length }`

    **In-source vitest tests:**

    Create a mock MCP server for testing by writing a small inline Node script that:
    - Reads lines from stdin
    - On initialize request: writes back a valid initialize response
    - On initialized notification: does nothing (notification, no response)
    - On tools/list request: writes back a response with 2 mock tools

    Use `spawn('node', ['-e', mockScript])` as the test server config.

    Test cases:
    1. Happy path: listMcpTools with mock server returns 2 tools with correct names
    2. Timeout: listMcpTools with a server that never responds (sleep script) rejects within timeout
    3. measureMcpTokens returns tokens > 0 with confidence 'measured'
    4. Non-stdio transport: listMcpTools with { type: 'http' } rejects with transport error message
    5. Bad command: listMcpTools with { command: 'nonexistent-binary-xyz' } rejects with error

    **Important implementation details:**
    - NEVER use execSync -- all async with spawn
    - Always kill child process in finally/cleanup (prevent zombie processes)
    - readline interface for parsing newline-delimited JSON from stdout
    - Filter out non-JSON lines from stdout (some servers print startup messages)
    - Handle stderr gracefully (log if verbose, don't crash)
