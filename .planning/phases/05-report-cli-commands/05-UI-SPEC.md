# Phase 5 -- UI Design Contract

> Visual and interaction contract for CLI terminal output. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Medium:** Terminal (CLI). All visual output is plain text + ANSI escape codes rendered to stdout. No browser, no React, no CSS. The "design system" is column alignment, box-drawing characters, emoji branding, and picocolors.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (CLI project -- no shadcn) |
| Preset | not applicable |
| Component library | cli-table3 ^0.6.5 (terminal tables) |
| Icon library | Unicode emoji (system font) |
| Font | Terminal monospace (user's terminal emulator default) |
| Color library | picocolors ^1.1.1 (non-table ANSI colors) |

---

## Spacing Scale

Terminal spacing is measured in character cells, not pixels. All values are character counts.

| Token | Value | Usage |
|-------|-------|-------|
| inline | 1 char | Separator between inline elements (e.g., `Defined: 140`) |
| field-gap | 3 chars | Gap between column-aligned fields in summary rows |
| indent | 2 chars | List item indent in top-5 section and detail views |
| section-break | 1 blank line | Between logical sections (header, summary, top-5, footer) |
| major-break | 2 blank lines | Not used -- keep output compact for screenshot sharing |

Exceptions: cli-table3 manages its own internal cell padding (1 char default). Do not override `colAligns` or `colWidths` unless content overflows. Use `wordWrap: true` on all tables.

---

## Typography

Terminal output has no font control. "Typography" translates to text formatting conventions.

| Role | Format | Weight Equivalent | Line Height |
|------|--------|-------------------|-------------|
| Header | `{emoji} {Title} -- {Window}` + `{divider}` on next line | Bold via picocolors `bold()` | 1 line + divider line |
| Summary row | `Category    Defined: N   Used: N   Ghost: N   ~Xk tokens/session` | Regular (no ANSI) | 1 line per category |
| Top-5 item | `  N. name       ~Xk tokens  (category, last-used)` | Regular (no ANSI) | 1 line per item |
| Section label | `{emoji} {Label}:` | Bold via picocolors `bold()` | 1 line |
| Footer score | `Health: NN/100 (Grade)` | Bold + color via picocolors | 1 line |
| Footer hint | `See per-item details: ccaudit inventory` | Dim via picocolors `dim()` | 1 line per hint |
| Table header | cli-table3 `head` array | Cyan via cli-table3 `style.head: ['cyan']` | Managed by cli-table3 |
| Table cell | cli-table3 cell content | Regular (no ANSI unless semantic) | Managed by cli-table3 |

**Text casing rules:**
- Category names: Title Case in headers, lowercase in summary rows (`Agents` in section headers, `agent` in data)
- Grade labels: Title Case always (`Healthy`, `Fair`, `Poor`, `Critical`)
- Tier labels: UPPERCASE in brackets (`[GHOST]`, `[LIKELY]`, `[ACTIVE]`)
- Recommendation labels: Title Case (`Archive`, `Monitor`, `Keep`)

## Color

Color is ANSI terminal color via picocolors. cli-table3 manages its own table header color.

| Role | ANSI Color | Usage |
|------|------------|-------|
| Dominant (60%) | Default terminal foreground | All body text, summary rows, data values |
| Secondary (30%) | `dim` (picocolors) | Footer hints, scan metadata, verbose output |
| Accent (10%) | `cyan` (picocolors + cli-table3) | Table headers, section dividers, the `--since` window value |
| Health: Healthy | `green` (picocolors) | Score display when grade is Healthy (>=80) |
| Health: Fair | `yellow` (picocolors) | Score display when grade is Fair (>=50) |
| Health: Poor | `red` (picocolors) | Score display when grade is Poor (>=20) |
| Health: Critical | `red` + `bold` (picocolors) | Score display when grade is Critical (<20) |
| Ghost tier label | `red` (picocolors) | `[GHOST]` tier label in detail views |
| Likely tier label | `yellow` (picocolors) | `[LIKELY]` tier label in detail views |
| Active tier label | `green` (picocolors) | `[ACTIVE]` tier label in detail views |
| Recommendation: Archive | `red` (picocolors) | Archive recommendation in detail table cells |
| Recommendation: Monitor | `yellow` (picocolors) | Monitor recommendation in detail table cells |
| Recommendation: Keep | `green` (picocolors) | Keep recommendation in detail table cells |

**Accent reserved for:** cli-table3 table header row text, the `--since` window value in output headers, and the heavy box-drawing divider character.

**NO_COLOR compliance:** picocolors auto-detects `NO_COLOR` and `FORCE_COLOR` environment variables. cli-table3 color is applied via picocolors wrappers that respect the same env vars. Phase 6 formalizes `--no-color` flag; Phase 5 relies on picocolors auto-detection only.

## Layout Contract

### Default Ghost Command (`npx ccaudit` / `ccaudit ghost`)

Source: `docs/ccaudit-handoff-v6.md` lines 228-240, CONTEXT.md D-01 through D-10.

```
{blank line}
{emoji} {Title} -- {since window}              # Header line (bold)
{heavy box divider: 32+ chars}                  # Divider line (cyan)
{blank line}
{category}    Defined: N   Used: N   Ghost: N   ~Xk tokens/session    # Summary rows (4 lines, column-aligned plain text)
{category}    Defined: N   Used: N   Ghost: N   ~Xk tokens/session
{category}    Defined: N   Used: N   Ghost: N   ~Xk tokens/session
{category}    Loaded:  N   Active: N  Stale: N   ~Xk tokens/session   # Memory uses domain labels (D-04)
{blank line}
Total ghost overhead: ~Xk tokens/session (X% of 200k window)          # Total line
{blank line}
{emoji} Top ghosts by token cost:                                       # Top-5 header (bold)
  1. {name}       ~Xk tokens  ({category}, {last-used})                # Top-5 items (max 5, sorted by token desc)
  2. {name}       ~Xk tokens  ({category}, {last-used})
  ...
{blank line}
Health: NN/100 ({Grade})                                                # Score line (colored by grade)
{blank line}
See per-item details: ccaudit inventory                                 # Hint 1 (dim)
Dry-run coming in v1.1: npx ccaudit@latest --dry-run                   # Hint 2 (dim)
```

**Locked layout decisions (from CONTEXT.md):**
- D-01: Summary-row layout (one row per category) + top-5 ghosts section below
- D-02: Summary rows are column-aligned plain text, NOT cli-table3 borders
- D-03: Top-5 is numbered plain-text list sorted by token cost descending
- D-04: Memory files use `Loaded/Active/Stale` labels; all others use `Defined/Used/Ghost`
- D-06: Consistent branding: emoji + title + since window on line 1, heavy divider on line 2
- D-07: Emoji mapping: ghost=`U+1F47B`, inventory=`U+1F4E6`, mcp=`U+1F50C`, trend=`U+1F4C8`
- D-08: Header format: `{emoji} {Title} -- Last {N} days` (em dash, human-readable window)
- D-09: Footer: health score line + two hint lines (inventory hint + v1.1 dry-run hint)
- D-10: Top-5 section header uses `U+1F6A8` emoji

**Summary row column alignment:**
- Category name column: left-aligned, padded to 13 chars (longest = "Memory Files")
- `Defined: N` / `Loaded: N`: right-align the number to 3 chars
- `Used: N` / `Active: N`: right-align the number to 3 chars
- `Ghost: N` / `Stale: N`: right-align the number to 3 chars
- `~Xk tokens/session`: right-aligned

**Heavy box divider:** Use Unicode `U+2501` (BOX DRAWINGS HEAVY HORIZONTAL) repeated to match header width. Minimum 32 characters.

### Inventory Command (`ccaudit inventory`)

Uses cli-table3 bordered table. Column set:

| Column | Width | Align | Content |
|--------|-------|-------|---------|
| Name | auto | left | Item name |
| Category | auto | left | agent / skill / mcp-server / memory |
| Scope | auto | left | global / project |
| Tier | auto | center | GHOST / LIKELY / ACTIVE |
| Last Used | auto | right | `Nd ago` or `never` |
| ~Token Cost | auto | right | `~Xk tokens (confidence)` |
| Action | auto | center | Archive / Monitor / Keep |

Header: `U+1F4E6 Inventory -- Last {N} days` with heavy divider.
Footer: Health score line (same format as ghost command).

### MCP Command (`ccaudit mcp`)

Uses cli-table3 bordered table. Column set:

| Column | Width | Align | Content |
|--------|-------|-------|---------|
| Server | auto | left | MCP server name |
| Scope | auto | left | global / project |
| Tier | auto | center | GHOST / LIKELY / ACTIVE |
| Invocations | auto | right | Count in window |
| Last Used | auto | right | `Nd ago` or `never` |
| ~Token Cost | auto | right | `~Xk tokens (confidence)` |
| Action | auto | center | Archive / Monitor / Keep |

Header: `U+1F50C MCP Servers -- Last {N} days` with heavy divider.
Mode indicator: `(estimated)` or `(live)` appended to header when `--live` flag used.
Footer: Total MCP overhead line + health score line.

### Trend Command (`ccaudit trend`)

Uses cli-table3 bordered table. Column set:

| Column | Width | Align | Content |
|--------|-------|-------|---------|
| Period | auto | left | ISO date (daily) or `Week of YYYY-MM-DD` (weekly) |
| Agents | auto | right | Invocation count |
| Skills | auto | right | Invocation count |
| MCP | auto | right | Invocation count |
| Total | auto | right | Sum |

Header: `U+1F4C8 Invocation Trend -- Last {N} days` with heavy divider.
Granularity auto-selection: daily for `--since <=7d`, weekly for `--since >7d` (Claude's discretion from CONTEXT.md).
Zero-fill: Empty time buckets show `0`, not blank cells.

## Branding Contract

Emoji usage is intentional branding, not decoration (source: handoff doc lines 35-41, CONTEXT.md D-07).

| Emoji | Unicode | Context | Meaning |
|-------|---------|---------|---------|
| `U+1F47B` | ghost | Default command header, ghost references in copy | Core brand identity |
| `U+1F4E6` | package | Inventory command header | Full inventory view |
| `U+1F50C` | electric plug | MCP command header | MCP server focus |
| `U+1F4C8` | chart increasing | Trend command header | Frequency over time |
| `U+1F6A8` | rotating light | Top-5 ghosts section header | Urgency / attention |

**Viral screenshot optimization:** The default ghost command output (summary + top-5 + health score) is designed to be screenshot-shared. Keep it to 20-25 lines maximum so it fits in a single terminal screenshot without scrolling.

## JSON Output Contract

When `--json` flag is active, skip ALL table rendering and emit raw JSON to stdout. Never mix ANSI escape codes with JSON output. The JSON structure is the stable API contract; text output format may change between versions.

**Ghost command JSON shape** (already established in ghost.ts):
```json
{
  "window": "7d",
  "files": 42,
  "projects": 3,
  "inventory": 200,
  "ghosts": { "total": 128, "likely": 40, "definite": 88 },
  "totalOverhead": { "tokens": 109000, "percentage": "54.5", "contextWindow": 200000 },
  "healthScore": { "score": 42, "grade": "Poor", "ghostPenalty": 48, "tokenPenalty": 10 },
  "items": [...]
}
```

Phase 5 adds the `healthScore` object and per-item `recommendation` field to the existing JSON structure.

## Interaction States

This phase is read-only CLI output. No interactive states exist.

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading / scanning | Command invoked | No spinner in v1. Verbose mode prints scan progress lines. |
| Success: ghosts found | Scan complete, ghost count > 0 | Full report layout (summary + top-5 + score + hints) |
| Success: no ghosts | Scan complete, ghost count = 0 | Clean state message + score 100/100 |
| Success: no sessions | No session files found | Path guidance + longer window suggestion |
| Error: bad input | Invalid `--since` value | Error message to stderr, exit code 1 |
| Error: scan failure | File system / parse error | Error message to stderr, exit code 1 |
