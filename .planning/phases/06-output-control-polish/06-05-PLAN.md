<objective>
Gap #1 closure: fix the CI coverage invocation so thresholds are actually enforced, and ensure the threshold bar currently passes on HEAD.

Root cause: `.github/workflows/ci.yaml` line 48 runs `pnpm test -- --run --coverage --coverage.thresholds.lines=80 ...`. pnpm expands the root `test` script (`TZ=UTC vitest`) and passes the `--` through. vitest treats `--` as the positional-argument separator, so `--run`, `--coverage`, and every threshold flag become positional file filters. No coverage provider activates. No `coverage/` directory is created. No thresholds are enforced. All 283 tests pass and CI exits 0, silently skipping SC-6.

Fix: Move coverage configuration (provider, thresholds, reporter, exclude list) into `vitest.config.ts` so it is enforced regardless of invocation path. This is the preferred path from VERIFICATION.md `missing:` list (option 3) and aligns with the ccusage pattern of config-as-source-of-truth.

Change CI YAML to invoke coverage via `pnpm exec vitest --run --coverage` (direct exec, no `--` delimiter). Keep the root `test` script unchanged (still `TZ=UTC vitest`) so local `pnpm test` behavior is undisturbed.

Secondary fix: the current HEAD has several files below 80% branch / line coverage. The largest drag is `apps/ccaudit/src/cli/commands/*.ts` command runners, which are end-to-end tested via the subprocess-based integration test (`src/__tests__/ghost-command.test.ts` spawns `node dist/index.js`) and cannot be instrumented by v8 coverage — v8 only sees code loaded into the vitest worker process. Barrel `index.ts` files and pure `types.ts` files similarly show 0% because they contain zero executable code. These are excluded from coverage with written justification. Remaining below-threshold gaps in the terminal table renderers (branches in `mcp-table.ts`, `ghost-table.ts`, `inventory-table.ts`) are closed by adding focused branch tests for the `formatTier` / `formatRecommendation` / `formatLastUsed` helpers.

Purpose: Phase 6 SC-6 requires CI to fail when coverage drops below 80%. Without this fix, the phase-6 quality gate is a silent no-op.
Output: vitest.config.ts with coverage provider + thresholds + exclude list, fixed CI YAML invocation, added branch tests in three table renderers, documented exclusions.
