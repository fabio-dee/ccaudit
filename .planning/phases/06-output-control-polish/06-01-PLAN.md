    - csvRow(['a','b,c','d']) returns 'a,"b,c",d'
    - csvTable(['h1','h2'], [['a','b'],['c','d']], true) returns 'h1,h2\na,b\nc,d'

    - tsvRow(['a','b','c']) returns 'a\tb\tc'

  <action>
    Create three new modules in packages/terminal/src/:

    **color.ts** (per D-04, D-05, D-06):
    ```typescript
    import pc from 'picocolors';
    import { createColors } from 'picocolors';
    ```
    - `let colorEnabled: boolean | undefined;` module-level state
    - `initColor(noColorFlag: boolean): void` -- sets colorEnabled based on flag OR `process.env.NO_COLOR` (present and non-empty string disables color, per NO_COLOR spec). Called once per command invocation before any rendering.
    - `isColorEnabled(): boolean` -- returns colorEnabled if set, else falls back to `pc.isColorSupported`
    - `getTableStyle(): Record<string, unknown>` -- returns `{ head: ['cyan'] }` when color enabled, `{}` when disabled. This prevents cli-table3's @colors/colors from applying ANSI codes (Pitfall 1 from research).
    - `colorize` object: wraps picocolors functions (bold, cyan, red, yellow, green, dim) to return plain text when color disabled. Implementation: `const noColor = createColors(false);` then each method checks `isColorEnabled()` and delegates to `pc` or `noColor`. Export as: `export const colorize = { bold: (s) => isColorEnabled() ? pc.bold(s) : noColor.bold(s), ... }` -- note `createColors(false)` returns identity functions that pass strings through unchanged.
    - In-source vitest tests covering all behaviors listed above.

    **csv.ts** (per D-18, D-19):
    - `csvEscape(value: string): string` -- if value contains comma, double-quote, or newline, wrap in double quotes and double any internal quotes. Otherwise return as-is. Per RFC 4180.
    - `csvRow(fields: string[]): string` -- maps each field through csvEscape, joins with comma.
    - `csvTable(headers: string[], rows: string[][], includeHeader = true): string` -- if includeHeader, first line is csvRow(headers). Then each row. Joined with '\n'.
    - In-source vitest tests covering all behaviors listed above.

    **quiet.ts** (per D-09):
    - `tsvRow(fields: string[]): string` -- joins fields with '\t' (tab character). No escaping needed for tab-separated output.
    - In-source vitest test: `tsvRow(['a','b','c'])` returns `'a\tb\tc'`.
